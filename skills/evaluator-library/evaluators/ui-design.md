### Axes

| Axis | What it measures |
|------|------------------|
| `coherence` | Does this screen/component belong to one product system? Type scale, color palette, and motion vocabulary consistent with the system. Visual hierarchy communicates priority. Scope: SYSTEM-level alignment with the design language. |
| `originality` | Does this contribute a fresh idea, or is it a derivative pastiche of every dashboard you've ever seen? Aim for "memorable", not "novel-for-novelty's-sake". |
| `craft` | Are the system's tokens applied with precision? Alignment grid respected, no misaligned baselines, breathing room consistent, transitions non-jittery. Scope: EXECUTION quality of this specific artifact. |
| `functionality` | Does it accomplish the user goal in fewest steps? Are affordances discoverable? Does it survive being resized, mis-clicked, network-throttled? |

### Score anchors (each axis 1-5)

- **5 — Exemplary.** Portfolio-worthy. Could ship to public marketing site as-is.
- **4 — Solid.** Production-quality. Minor polish only.
- **3 — Acceptable.** Functional but unremarkable. Specific issues to address before launch.
- **2 — Needs work.** Visible defects, mis-aligned with system, or task-completion friction.
- **1 — Reject.** Fundamental redesign required.

### Critique guidance

For any axis scored ≤3, cite the specific screen / component / interaction. Note which design-system primitive (token, component) is wrong or missing. Avoid taste-only feedback; ground critique in the rubric anchors.
